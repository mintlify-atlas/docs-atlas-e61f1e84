title: "Directory Score"
description: "How the Directory Score is calculated for each library in React Native Directory."
---

The Directory Score is a 0–100 quality indicator assigned to every library in React Native Directory. It reflects the overall health and activity of a package based on data available from GitHub and npm.

<Warning>
  Directory scores are subjective and are based on data that's readily available on GitHub and npm. They may not reflect quality for every use case. When evaluating libraries for your project, review the library yourself to determine if it's a good fit.
</Warning>

## How it works

The score is built by evaluating a set of criteria against each library. Each criterion that applies either adds or subtracts points from the raw score. The raw score is then clamped to a final value between **0 and 100**.

A library can earn points by meeting "good behavior" criteria and lose points by meeting "bad behavior" criteria.

## Combined Popularity

Several scoring criteria depend on a derived metric called **Combined Popularity**, which weights forks, stars, and monthly npm downloads:

```typescript
forks * 20 + stars * 10 + downloads / 50
```

This single number is used as the threshold for the popularity-based criteria below.

## Scoring criteria

Each criterion is evaluated independently. All matching criteria are summed to produce the raw score.

<AccordionGroup>
  <Accordion title="Very popular (+40)">
    Libraries with a Combined Popularity score above 100,000 meet this criterion.
  </Accordion>

  <Accordion title="Popular (+20)">
    Libraries with a Combined Popularity score above 10,000 meet this criterion.
  </Accordion>

  <Accordion title="Known (+10)">
    Libraries with a Combined Popularity score above 2,500 meet this criterion.
  </Accordion>

  <Accordion title="Recently updated (+10)">
    Libraries that have been updated within the last 60 days meet this criterion.
  </Accordion>

  <Accordion title="Has a README file (+10)">
    Libraries that include a README file in their GitHub repository or npm package meet this criterion.
  </Accordion>

  <Accordion title="Has a description (+5)">
    Libraries that have a description defined in their `package.json` meet this criterion.
  </Accordion>

  <Accordion title="Has vulnerability alerts enabled (+5)">
    Libraries with vulnerability alerts enabled in their GitHub repository meet this criterion.
  </Accordion>

  <Accordion title="New Architecture support unknown (-5)">
    Libraries that do not have New Architecture support defined or confirmed by code analysis meet this criterion. This penalty does not apply to dev tools, templates, or Expo Go compatible packages.
  </Accordion>

  <Accordion title="Not updated recently (-10)">
    Libraries that have not been updated in the last 180 days meet this criterion.
  </Accordion>

  <Accordion title="Lots of open issues (-10)">
    Libraries with more than 100 open issues (if they have at least 1,000 stars), or more than 50 open issues otherwise, meet this criterion.
  </Accordion>

  <Accordion title="Has a lot of dependencies (-10)">
    Libraries with 25 or more direct dependencies meet this criterion.
  </Accordion>

  <Accordion title="Not supporting New Architecture (-15)">
    Libraries that explicitly do not support the New Architecture meet this criterion. This penalty does not apply to dev tools or templates.
  </Accordion>

  <Accordion title="No license, GPL license, or unrecognized license (-20)">
    Libraries without a license, with a GPL license, or with a non-standard license meet this criterion.
  </Accordion>

  <Accordion title="Unmaintained (-30)">
    Libraries that are archived on GitHub, deprecated on npm, or marked as unmaintained due to lack of activity in the last two years meet this criterion.
  </Accordion>
</AccordionGroup>

## Score range

The final Directory Score is always between **0 and 100**. Raw scores below 0 are clamped to 0, and raw scores above 100 are clamped to 100.

<Note>
  You can view the source code used to calculate Directory Scores in [`scripts/calculate-score.ts`](https://github.com/react-native-community/directory/blob/main/scripts/calculate-score.ts) and the criteria definitions in [`util/scoring.ts`](https://github.com/react-native-community/directory/blob/main/util/scoring.ts).
</Note>
